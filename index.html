Git - Система Контролю Версій

Коротка інформація про системи контролю версій та Git

Система контролю версій (СКВ) дозволяє зберігати різні версії файлів, відслідковувати зміни та управляти історією проекту. Git є однією з найпопулярніших систем контролю версій, яка була створена Лінусом Торвальдсом в 2005 році. Вона є розподіленою системою, що дозволяє зберігати копії проектів на різних пристроях.

Основні можливості та переваги використання Git

  • Розподілена архітектура, що дозволяє працювати без постійного доступу до інтернету.
  • Відстеження змін у файлах та зберігання історії проєкту.
  • Паралельна робота над проектом декількох розробників через гілки (branches).
  • Легке злиття (merge) змін з різних гілок.
  • Безпека даних за рахунок хешування кожної зміни.

10 базових команд Git з коментарями

  1. git init – Ініціалізує новий Git-репозиторій у поточній директорії.
  2. git clone [URL] – Клонує репозиторій з віддаленого сервера на локальний комп'ютер.
  3. git status – Виводить інформацію про стан робочого каталогу і індексу.
  4. git add [файл] – Додає файл до індексу для наступного коміту.
  5. git commit -m "[повідомлення]" – Створює новий коміт з доданими файлами.
  6. git push – Відправляє коміти з локального репозиторію на віддалений сервер.
  7. git pull – Оновлює локальний репозиторій, отримуючи останні зміни з віддаленого.
  8. git branch – Переглядає всі гілки в репозиторії.
  9. git checkout [гілка] – Перемикається на іншу гілку.
  10. git merge [гілка] – Зливає вказану гілку з поточною.
